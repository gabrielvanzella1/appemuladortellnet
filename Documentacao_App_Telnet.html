1. VISÃO GERAL DO PROJETO

Objetivo: Emulador Telnet avançado para Android, permitindo conexões a servidores Telnet remotos com suporte a cores ANSI, histórico de comandos, persistência de dados e licenciamento enterprise.

Público-alvo: Profissionais de TI, administradores de sistemas, desenvolvedores e entusiastas de tecnologia retrô.

Plataforma: Android 7.0+ (API 24), com interface nativa Kotlin + Android Jetpack.

🎯 Diferenciais:
  • Protocolo Telnet RFC 854 com suporte completo
  • Cores ANSI/VT100 em tempo real
  • Histórico inteligente de comandos
  • Licenciamento com device fingerprinting SHA-256
  • Banco de dados local robusto (Room v2)
  • Interface otimizada para mobile

2. FUNCIONALIDADES IMPLEMENTADAS ✅

2.1 - Cliente Telnet (RFC 854)

  • Socket TCP real com suporte completo ao protocolo Telnet RFC 854
  • Negociação de opções (WILL, WONT, DO, DON'T)
  • Leitura não-bloqueante com BufferedReader.ready()
  • Tratamento automático de conexões e desconexões
  • Suporte a diferentes portas (padrão 23)
  • Timeout configurável para conexões

2.2 - Terminal com Cores ANSI/VT100

  • Parser completo de sequências ANSI (\\x1b[...m)
  • Suporte a 16 cores básicas (30-37, 90-97)
  • Cores de background (40-47)
  • Estilos: bold, italic, underline, reverse
  • Renderização em HTML com WebView
  • Performance otimizada para grandes volumes de texto

2.3 - Histórico de Comandos

  • Navegação com setas ↑ (anteriormente) e ↓ (próximo)
  • Armazenamento em memória (máx 100 comandos)
  • InputHistoryManager para controle
  • Retenção durante sessão ativa
  • Limpeza automática ao desconectar

2.4 - Banco de Dados (Room v2)

  • Persistência local com SQLite
  • Entidades: SavedConnection, CommandHistory, SessionLog, AppPreference, LicenseInfo
  • Migrações automáticas com fallbackToDestructiveMigration
  • Operações assíncronas com Coroutines (Dispatcher.IO)
  • Foreign keys e constraints
  • DAOs com operações CRUD completas

2.5 - Conexões Salvas

  • Spinner dropdown com lista de hosts salvos
  • Auto-preenchimento de host/porta ao selecionar
  • Salvar novas conexões com nome customizável
  • Favoritos e histórico de último acesso
  • Edição de conexões existentes
  • Exclusão de conexões com confirmação

2.6 - Licenciamento Enterprise

  • Device Fingerprinting com SHA-256 (impossível falsificar)
  • Licenças TRIAL (30 dias) e PREMIUM (1 ano)
  • Validação por device vinculado
  • Menu 📜 Licença com dialog customizado
  • Armazenamento seguro em banco de dados
  • Expiração automática com validação

2.7 - Controle de Teclado Virtual

  • Auto-ocultar teclado ao conectar ao servidor
  • Auto-mostrar teclado ao desconectar
  • Ocultar após envio de comando
  • Melhor visualização do terminal
  • Controle via InputMethodManager

5. SEGURANÇA E LICENCIAMENTO

5.1 - Device Fingerprinting

  • Método: SHA-256 hash de hardware único
  • Componentes: ANDROID_ID + Model + Manufacturer + Serial + Fingerprint
  • Segurança: Impossível falsificar sem mudança física do hardware
  • Formato: 32 caracteres hexadecimais
  • Colisão: Praticamente impossível (SHA-256)

5.2 - Modelos de Licença

Tipo Duração Renovação Uso
TRIAL 30 dias Única vez Teste completo do app
PREMIUM 1 ano Anual Uso comercial/profissional
ENTERPRISE Ilimitado - Corporativo multi-device

5.3 - Validação de Licença

  • Verifica device fingerprint na ativação
  • Valida data de expiração em tempo real
  • Armazenamento seguro em banco local (criptografia futuro)
  • Sincronização com servidor (fase 10)
  • Detecção de modificações não autorizadas

5.4 - Proteção de Dados

  • Senhas não armazenadas localmente
  • Histórico de comandos criptografado (fase 10)
  • Conexões via HTTPS/TLS 1.2+ (futuro)
  • Certificação SSL/TLS obrigatória em SSH
  • EncryptedSharedPreferences para dados sensíveis
  • Rate limiting de tentativas de login

6. AUTENTICAÇÃO (LOGIN/CADASTRO)

6.1 - Fluxo de Login

Tela de Login:

  1. Usuário insere email e senha
  2. Validação local (email válido, senha não vazia)
  3. Envio para servidor (HTTPS POST com TLS 1.2+)
  4. Servidor retorna token JWT ou erro 401/403
  5. Token armazenado localmente (EncryptedSharedPreferences)
  6. Redirecionamento para MainActivity
  7. Sessão persiste até logout ou expiração

6.2 - Fluxo de Cadastro de Usuário

Tela de Registro (Multi-step):

  1. Usuário preenche formulário (nome, email, senha)
  2. Validação de força de senha (8+ chars, uppercase, números, símbolos)
  3. Verificação de email único (HTTPS POST)
  4. Envio de confirmação por email com token
  5. Usuário clica link no email (expira em 24h)
  6. Confirmação de email concluída
  7. Redirecionamento para login
  8. Login + redirect para app

6.3 - Recuperação de Senha

  • Tela "Esqueci a senha?" acessível do login
  • Usuário insere email cadastrado
  • Verificação de existência do usuário
  • Email com link de reset enviado
  • Token com expiração de 1 hora
  • Tela para inserir nova senha
  • Confirmação via email
  • Redirecionamento para login

6.4 - Segurança de Senhas

  • Mínimo 8 caracteres
  • Obrigatório: 1 maiúscula, 1 minúscula, 1 número, 1 símbolo
  • Hashing: bcrypt+salt no servidor (nunca plaintext)
  • Transmissão: HTTPS TLS 1.2+
  • Rate limiting: Máx 5 tentativas falhadas / 5 minutos
  • 2FA: Suporte futuro (SMS/Google Authenticator)

7. CADASTRO DE EMPRESA

7.1 - Requisitos de Cadastro

  • Usuário: Deve estar logado
  • Permissão: Apenas usuário com role 'ADMIN' ou 'OWNER'
  • CNPJ: Validado contra banco de dados (único)
  • Confirmação: Email de verificação da empresa
  • Plano: Seleção obrigatória (TRIAL/PREMIUM)

7.2 - Fluxo de Cadastro Empresarial

Passo 1 - Dados Básicos:

  • Razão social (obrigatório, máx 255 chars)
  • CNPJ (validação automática de formato e DV)
  • Inscrição estadual (opcional)
  • Classificação (Indústria, Comércio, Serviços, etc)

Passo 2 - Contato:

  • Email corporativo (validado)
  • Telefone (formato internacional +55)
  • Website (opcional, validação de URL)

Passo 3 - Endereço:

  • Rua e número (obrigatório)
  • Complemento (opcional)
  • CEP (auto-preenche cidade/estado via ViaCEP)
  • Cidade e estado
  • País (padrão Brasil)

Passo 4 - Documentos:

  • Upload de logo (PNG/JPG, máx 5MB)
  • Dimensões mín 200x200px, recomendado 512x512px
  • Descrição da empresa (até 500 caracteres)

Passo 5 - Plano e Pagamento:

  • Seleção de plano (TRIAL 30d, PREMIUM 1 ano)
  • Método de pagamento (Cartão, PIX, Boleto)
  • Resumo com valores e taxas
  • Aceitar termos de serviço
  • Confirmação de criação

7.3 - Validações

  • CNPJ: Validação via algoritmo de dígito verificador
  • CEP: Consulta API de endereços (ViaCEP/Postmon)
  • Email: Validação de domínio corporativo + envio de confirmação
  • Telefone: Formato internacional (+55 11 XXXXX-XXXX)
  • Logo: Validação de tipo de arquivo, tamanho, dimensões
  • Unicidade: Verificação de CNPJ duplicado no BD

7.4 - Após Cadastro

  • Email de confirmação com código de verificação
  • Usuário criador vira OWNER da empresa
  • Pode adicionar outros usuários (ADMIN, USER, VIEWER)
  • Ativa licença TRIAL (30 dias) ou PREMIUM (1 ano)
  • Acesso ao painel administrativo
  • Visibilidade em API de empresas (listagem)

CONCLUSÃO

O Emulador Telnet é um aplicativo Android robusto e profissional que combina funcionalidades clássicas de terminal com segurança moderna e licenciamento enterprise. Com a arquitetura MVVM e uso de Room Database, o app oferece performance, confiabilidade e escalabilidade para usuários exigentes.

As próximas fases focam em autenticação, gestão de usuários/empresas e um ecosistema completo de plataforma, mantendo o foco em segurança, escalabilidade e experiência do usuário.

Com a conclusão da Phase 6 (Licenciamento Enterprise), o app está preparado para publição na Google Play Store e pronto para receber usuários pagos e empresas.
